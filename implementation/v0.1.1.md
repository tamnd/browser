# v0.1.1: daily drivable

Goal: the features you notice missing within an hour of real browsing.
This is the "daily drivable" milestone, released as a patch bump per project convention.

## Checklist

- [x] Per-workspace containers: each workspace gets its own WKWebsiteDataStore(forIdentifier:), persisted in the session snapshot
- [x] Content blocking: blocklist.json in the profile compiled through WKContentRuleListStore, third-party block rules per domain, allowlist support, shields.toggle and shields.site commands
- [x] Downloads: WKDownloadDelegate into the configured folder with name dedupe, progress in the sidebar, downloads.reveal and downloads.clear commands
- [x] Find in page: cmd+f bar with incremental search, cmd+g and cmd+shift+g cycle matches
- [x] Favicons: fetched once per host, cached in profile favicons/, shown in tab rows
- [x] Closed tab undo: bounded stack of 50, tab.reopen on cmd+shift+t restores into the original workspace
- [x] Webview cap: tabs.maxLiveWebviews config knob (default 12), least recently used background webview is dropped and reloads on focus
- [x] Per-site zoom: zoom changes persist per host in SQLite and reapply on load
- [x] Error pages: readable inline page on load failure instead of a blank view
- [x] Config additions: privacy, tabs, and downloads sections with tolerant decoding
- [x] Tests for the blocker JSON, closed tab stack, site zoom migration, error page escaping, favicon filenames, download name dedupe, container round trip
- [x] Docs: README feature list and profile README updated

## Cut from this release

- Multiwindow.
  The app is built around one shared AppModel and one key monitor, and splitting that cleanly is its own release.
- Idle-timer tab suspension.
  The LRU webview cap already bounds memory, a timer on top adds config surface without much gain yet.
- Download resume from resumeData.
  Failed downloads just report failure for now.

## Notes

- Containers ride on `WKWebsiteDataStore(forIdentifier:)` (macOS 14+), keyed by a `containerID` UUID on each workspace.
  The id persists through the session snapshot, so cookies survive relaunch.
  Sessions saved by v0.1.0 have no container id and get a fresh one, which means a one-time cookie reset per workspace.
  Acceptable pre-release, noted here so nobody hunts for a bug later.
- Block rules are generated, not hand-written: one `^https?://([^:/]+\.)?domain[:/]` third-party rule per domain, then raw rules from the file, then one `ignore-previous-rules` per allowlisted site.
  Rule order matters to WebKit, allowlist entries must come last.
- The shields.toggle and shields.site commands are runtime-only state.
  A config or blocklist reload rebuilds rules from the files, which is the documented source of truth.
- Downloads come from two WebKit paths: `navigationAction.shouldPerformDownload` (anchor download attribute) and `canShowMIMEType == false` on the response.
  WebKit fires "frame load interrupted" (WebKitErrorDomain 102) when a navigation turns into a download, so the error page handler swallows that code.
- Find uses `WKWebView.find(_:configuration:)`, which selects the match natively.
  The find bar is a lightweight card, not a focus-stealing overlay, so typing stays in its TextField and escape is handled by the key monitor.
- The LRU cap evicts only webviews not visible in the active workspace's panes.
  Eviction drops the WKWebView but keeps the Tab value, so the page reloads from its URL on next focus.
  Back-forward history of evicted tabs is lost; a session-restore-style archive could fix that later.
- Per-site zoom stores one row per host and deletes the row at zoom 1.0, so the table only holds exceptions.
- Favicons are a plain `https://host/favicon.ico` fetch, one attempt per host per run, cached on disk.
  Parsing HTML for `<link rel=icon>` is deliberately out of scope this release.
- 19 new tests, 51 total.
