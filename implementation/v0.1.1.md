# v0.1.1: daily drivable

Goal: the features you notice missing within an hour of real browsing.
This is the "daily drivable" milestone, released as a patch bump per project convention.

## Checklist

- [ ] Per-workspace containers: each workspace gets its own WKWebsiteDataStore(forIdentifier:), persisted in the session snapshot
- [ ] Content blocking: blocklist.json in the profile compiled through WKContentRuleListStore, third-party block rules per domain, allowlist support, shields.toggle and shields.site commands
- [ ] Downloads: WKDownloadDelegate into the configured folder with name dedupe, progress in the sidebar, downloads.reveal and downloads.clear commands
- [ ] Find in page: cmd+f bar with incremental search, cmd+g and cmd+shift+g cycle matches
- [ ] Favicons: fetched once per host, cached in profile favicons/, shown in tab rows
- [ ] Closed tab undo: bounded stack of 50, tab.reopen on cmd+shift+t restores into the original workspace
- [ ] Webview cap: tabs.maxLiveWebviews config knob (default 12), least recently used background webview is dropped and reloads on focus
- [ ] Per-site zoom: zoom changes persist per host in SQLite and reapply on load
- [ ] Error pages: readable inline page on load failure instead of a blank view
- [ ] Config additions: privacy, tabs, and downloads sections with tolerant decoding
- [ ] Tests for the blocker JSON, closed tab stack, site zoom migration, error page escaping, favicon filenames, download name dedupe, container round trip
- [ ] Docs: README feature list and profile README updated

## Cut from this release

- Multiwindow.
  The app is built around one shared AppModel and one key monitor, and splitting that cleanly is its own release.
- Idle-timer tab suspension.
  The LRU webview cap already bounds memory, a timer on top adds config surface without much gain yet.
- Download resume from resumeData.
  Failed downloads just report failure for now.
