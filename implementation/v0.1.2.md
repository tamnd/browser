# v0.1.2: command surface and plugin core

Goal: the first slice of the customization milestone.
One palette for everything, keymaps that can express multi-key sequences, and a plugin API that reacts to the browser instead of just poking it.

## Checklist

- [ ] Unified palette: one overlay behind cmd+l and cmd+k, mode picked by prefix
- [ ] Bare text navigates or searches with the existing omnibox heuristics
- [ ] `>` prefix lists commands with fuzzy match and keybinding hints
- [ ] `#` prefix searches tabs across all workspaces and jumps on enter
- [ ] cmd+l prefills the current URL, cmd+k prefills `>`, tab.search command prefills `#`
- [ ] Multi-key sequences in keymap.json, space separated like "g t", with a pending-keys indicator
- [ ] Sequence timeout resets pending keys, and bare-key sequences never swallow typing in pages or fields
- [ ] Prefix conflicts (a binding that is also the start of a longer one) reported as keymap warnings
- [ ] Plugin manifest v1: minAppVersion and permissions validated at load, unknown permissions rejected
- [ ] Plugin suggestedKeymap layered between defaults and user keymap, user always wins
- [ ] browser.events.on for tab.created, tab.closed, tab.activated, workspace.switched, navigation.committed, download.finished
- [ ] browser.styles.register and unregister for dynamic CSS, optional host filter
- [ ] Per-plugin unload and reload, with a Reload Plugin command per loaded plugin
- [ ] Plugin dev mode: plugins.devMode config flag hot-reloads enabled plugins on file change
- [ ] hello example plugin rewritten against the new API
- [ ] Tests for sequence parsing, keymap matching and layering, palette modes, manifest validation, style host filter
- [ ] Docs: README and profile README updated

## Cut from this release

- browser.fetch and the host permission prompts.
  Permission storage and UI deserve their own release.
- Plugin panels in isolated webviews.
- Internal pages (browser://settings, browser://history), reader mode, and the element zapper.
  Each is a patch release of its own.
