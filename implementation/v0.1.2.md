# v0.1.2: command surface and plugin core

Goal: the first slice of the customization milestone.
One palette for everything, keymaps that can express multi-key sequences, and a plugin API that reacts to the browser instead of just poking it.

## Checklist

- [x] Unified palette: one overlay behind cmd+l and cmd+k, mode picked by prefix
- [x] Bare text navigates or searches with the existing omnibox heuristics
- [x] `>` prefix lists commands with fuzzy match and keybinding hints
- [x] `#` prefix searches tabs across all workspaces and jumps on enter
- [x] cmd+l prefills the current URL, cmd+k prefills `>`, tab.search on cmd+shift+k prefills `#`
- [x] Multi-key sequences in keymap.json, space separated like "g t", with a pending-keys indicator
- [x] Sequence timeout resets pending keys, and bare-key sequences never swallow typing in pages or fields
- [x] Prefix conflicts (a binding that is also the start of a longer one) reported as keymap warnings
- [x] Plugin manifest v1: minAppVersion and permissions validated at load, unknown permissions rejected
- [x] Plugin suggestedKeymap layered between defaults and user keymap, user always wins
- [x] browser.events.on for tab.created, tab.closed, tab.activated, workspace.switched, navigation.committed, download.finished
- [x] browser.styles.register and unregister for dynamic CSS, optional host filter
- [x] Per-plugin unload and reload, with a Reload Plugin command per loaded plugin
- [x] Plugin dev mode: plugins.devMode config flag hot-reloads enabled plugins on file change
- [x] hello example plugin rewritten against the new API
- [x] Tests for sequence parsing, keymap matching and layering, palette modes, manifest validation, style host filter
- [x] Docs: README and profile README updated

## Notes

- The omnibox and the command palette were separate overlays with separate state.
  They are now one overlay whose mode comes from the first character of the input, so switching modes is just typing or deleting a prefix character.
  The old command ids (omnibox.open, palette.open) survive and just prefill differently.
- Tab search (`#`) matches against workspace name, tab title, and URL, so "#work docs" style queries narrow by workspace naturally.
  Committing a tab row switches workspace first, then selects the tab.
- Keymap bindings are now sequences of chords; a plain chord is a one-element sequence, so every existing keymap file keeps working.
  Matching is exact-first: an exact hit fires immediately, which makes any longer binding that shares the prefix unreachable.
  That conflict is reported as a load warning instead of being silently resolved.
- A bare key (no cmd/ctrl/alt) that only starts a sequence is passed through when a webview or text field has focus, so sequences never eat typing.
  Defaults ship with no sequences; they are an opt-in for user keymaps and plugin suggestions.
- The pending-keys chip sits bottom right and clears after 1.5 seconds of silence or any non-matching key.
- Keymap layering is defaults, then plugin suggestedKeymap entries, then the user file.
  A user null unbinds even a plugin suggestion.
- Manifest validation rejects any permission name because none exist yet; the first real one lands with browser.fetch.
  Old two-field manifests still load, minAppVersion is compared numerically part by part.
- Plugin events are fire-and-forget, synchronous JS calls on the main thread.
  navigation.committed fires from didFinish, so it means "page settled", not "request started".
- Plugin styles ride the same shared WKUserContentController as snippets, so they apply on the next navigation, not retroactively.
  The host filter runs in the injected script itself: exact host or subdomain suffix match.
- Dev mode watches each enabled plugin folder with the existing DirectoryWatcher (300ms debounce) and reloads all enabled plugins on change.
  A reload tears down the JSContext, so plugin state lives in browser.storage or dies.
- Plugins now unload on quit through per-plugin unload, which runs onunload, unregisters commands, drops listeners, and removes styles.
  Before, unloadAll existed but nothing called it.
- 21 new tests, 72 total.
